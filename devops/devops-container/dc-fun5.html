# Harbor 概述 - 官网

harbor 介绍

github 地址： https://github.com/goharbor/harbor

Habor是由VMWare公司开源的容器镜像仓库。事实上，Habor是在Docker Registry上进行了相应的 企业级扩展，从而获得了更加广泛的应用，这些新的企业级特性包括：管理用户界面，基于角色的访 问控制 ，AD/LDAP集成以及审计日志等，足以满足基本企业需求。

# Haobor 部署

# 部署方式

Harbor 可以通过以下两种方法之一安装：

  • 安装程序：安装程序从Docker hub下载Harbor的图像。因此，安装程序的尺寸非常小。
  • 离线安装程序：当主机没有Internet连接时使用此安装程序。安装程序包含预先构建的图像，因此其大小更大。
  • OVA安装程序：当用户具有vCenter环境时，使用此安装程序，在部署OVA后启动Harbor

所有安装程序都可以从官方发布页面下载。

本指南介绍使用联机或脱机安装程序安装和配置Harbor的步骤。安装过程几乎相同。

如果运行以前版本的Harbor，则可能需要更新harbor.yml和迁移数据以适应新的数据库架构。有关更多详细信息，请参阅“ 港口迁移指南”。

此外，社区还创建了Kubernetes的部署说明。有关详细信息，请参阅Harbour on Kubernetes。

# 目标主机的先决条件

Harbor被部署为多个Docker容器，因此可以部署在任何支持Docker的Linux发行版上。目标主机需要Docker，并且要安装Docker Compose。

# 硬件

资源 容量 描述
处理器 最小2 CPU 4 CPU是首选
内存 最小4GB 8GB是首选
磁盘 最小40GB 160GB是首选

# 软件

软件 版 描述
Docker引擎 版本17.03.0-ce +或更高版本 有关安装说明，请参阅：docker engine doc
Docker撰写 版本1.18.0或更高版本 有关安装说明，请参阅：docker compose doc
OpenSSL的 最新的是首选 为Harbor生成证书和密钥

# 网络端口

港口 协议 描述
443 HTTPS Harbor端口和核心API将接受此端口上的https协议请求，此端口可以在配置文件中更改
4443 HTTPS 只有在启用“公证”时才需要连接到Dock的Docker Content Trust服务，此端口可以在配置文件中更改
80 HTTP Harbor端口和核心API将接受此端口上的http协议请求

# 配置Harbor文件

注意，ssl不是必须配置的。主要看公司业务的具体情况

vim /opt/harbor/harbor.cfg
hostname = harbor.joonhee.com
ssl_cert = ./ssl/joonhee.crt
ssl_cert_key = ./ssl/joonhee.key
harbor_admin_password = Harbor12345 //默认密码，登陆后更改
:wq
./prepare
./install.sh
docker-compose ps //可以查看到harbor环境已经起来了
