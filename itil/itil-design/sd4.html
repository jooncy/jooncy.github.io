# 第五章：能力管理

# 目的/目标

  • 保证IT服务和IT基础设施的能力足以达到服务级别目标
  • 考虑提供IT服务需要的所有资源

# Objectives

  • 提出和维护适当的、最新的能力计划
  • 在其他业务和IT关于能力性能方面提供指导
  • 保证业务性能达到或者超过达成的性能目标
  • 评估能力计划、性能、服务及资源能力变更带来的影响
  • 保证主动度量，提高服务实施的性能

# 能力管理范围

  • 所有硬件
  • 所有网络设备
  • 所有外围设备
  • 所有的软件
  • 人力资源

# 定义三个子流程

  • 业务能力管理
    • 需求管理，预测筹划和记录未来需求
  • 服务能力管理
    • 工作量管理类，监控缝隙报告服务性能，建立工作量基线
  • 组件能力管理
    • 资源管理，管理系统和员工
    • 性能管理，调优和工作平衡

# 平衡能力管理

  • 成本 - 资源
  • 供应 - 需求

平衡

确保IT能力方面的花费是合理正当的

并且使IT能力能够符合当前及未来一明确的业务需求

# 重复性活动

持续化改进，一般开始的设计无需太复杂
