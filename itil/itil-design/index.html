# 序 前言

服务设计

负责设计心得或者变更的服务，引入生成环境。包括架构、流程、政策及文档. 包括一系列的流程和职能 各种管理的设计，对所有服务设计采取统一的策略。

  • 整个IT技术范围内的所有活动确保一致性和完整性
  • 提供端到端的 业务相关的功能和质量

# 服务设计流程

# 服务设计的价值

  • 主要价值
    • 确保业务需求在成本效益模式下，不降低质量的前提下被满足
  • 附加价值
    • 总有用成本
    • 改良的服务质量
    • 改良的服务一致性
    • 抑郁实现新的或者变更的服务
    • 改良的服务对其
    • 更有效的服务性能
    • 改良的IT管控
    • 改良的信息及决策

# 设计的范围

  • 新的或者变更的服务
  • 服务管理系统和工具，尤其是服务组合
  • 技术架构与管理系统
  • 必要的流程
  • 度量方法和度量指标

# 服务设计的交付方式

  • 内购
  • 外购
  • 合作伙伴或多方外包
  • 业务流程外包
  • 应用服务提供
  • 知识流程外包
