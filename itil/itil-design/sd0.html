# 第一章：服务目录管理

# 服务设计

  • Purpose
    • 定义一个关于所有被认可的服务单独的、一致的信息源并确保所有有权访问的人可用
  • Goal
    • 确保服务目录被制造和维护，包含准确的信息
      • 当前云个性的所有服务
      • 所有即将投入运行的服务
  • Objectives
    • 管理服务目录中包含的信息
    • 确保正式环境中正在运行和准备运行的服务准确，反应当前的细节、状态、接口和依赖关系

# 目录

# 目录类型

  • 业务目录，务流程与服务直接的关系
  • 服务目录，Email..
  • 技术目录，服务对应技术

# 服务目录经理

  • 服务目录经理负责生产和维护目录服务
  • 职责
    • 所有运行的服务，包括准备好运行的，记录在服务目录中
    • 服务目录中的所有信息是准确的和最新的
    • 服务目录中的所有信息与服务组合中的信息一致
    • 服务目录中的信息有足够的保护和备份
