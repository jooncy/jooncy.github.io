# 第二章：服务战略流程

# 管理类型

  • 服务组合管理
  • 财务管理
  • 需求管理

# 服务组合

  • 服务提供者管理的一个完整的服务集合
  • 包含了三个分类
    • 服务管道，建议或者开发中
    • 服务目录，在线或者可以部署
    • 退役服务
  • 服务组合管理
    • 负责管理服务组合的流程

# 产品经理角色

  • 职责
    • 在生命周期周，将服务作为一个产品来管理
    • 围绕服务目录来协调和关注组织
    • 与业务关系经理合作
    • 作为服务及服务目录的主题相关专家

# 财务管理

  • 负责管理IT服务提供者的如下财务需求
    • 预算
    • 会计核算
    • 收费

# 需求管理

  • 影响客户需求的活动
  • 满足这些需求的能力储备

# 定义业务活动模式

  • 客户资产 - 如：人员、流程、应用生成了业务活动模式（PBA）
  • 业务活动模式（PBA）
    • 产生预算、收入、成本
    • 核算大部分业务成果

# 跟踪PBA

  • 将类似的活动组合在一起，指定一个组编码
    • 只有非常不同的活动用一个单独的编码
  • 分类每个活动，按照一些要素
    • 是否为高频活动
    • 收入高或者低的活动
    • 该活动需要什么IT服务
  • PBA是降低复杂度和服务整体承包的三个“模块”
    • CPA
    • CSP
    • SLP
