# 第四章：服务管理中的流程

# 流程的定义

  • 在ITIL 服务生命周期中，流程是围绕着目标，利用流程的反馈在行动上进行自身的正面提升和就诊，流程以及只能都是闭环控制系统
    • 特定的目标（例如资源申请）
    • 确定的输入、处理后和输出
    • 角色、职责、工具及管理控制
    • 政策、标准、指导方针、活动及工作说明

# 流程的特点

  • 由特定的活动触发
  • 可以重复
  • 可管理
  • 可衡量，交付特定的结果
  • 有效
  • 有效率，用最小的努力和成本

# 流程控制

  • 计划和调节流程中的步骤的过程，以有效和高质行流程为目标
  • 流程所有者及流程管理人员
  • 流程模型
    • 形成文件的工作量，对执行流程提供指导
    • 有助于理解及清晰地描述流程雨中不同的特性
  • 戴明环

# 流程属主及管理者角色

  • 流程管理者
    • 负责流程的操作类管理
    • 规划实施和监控流程需要的活动
    • 一个流程可能有多个管理者
    • 通常是流程的属主
  • 流程管属主
    • 确保一个流程符合目标
    • 发起和设计流程
    • 确保适当的流程文档可用并且最新
    • 定义适当的正常和标准
    • 定期审核流程

# 流程、只能及角色

  • 流程
    • 为完成一个特定的目标，而设计的结构和的活动集合
    • 流程可以包含为保证可靠输出而需要的所有角色，职责，工具和管理控制
  • 智能
    • 一组人员的团队和工具，来完成一个或多个流程或者活动
  • 角色
    • 一个职责、活动和给一个人和团队的授权的集合
    • 角色在流程中定义
