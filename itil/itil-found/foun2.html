# 第二章：ITIL 概述

# ITIL的目的

  • 为IT服务管理提供“Good Practice” 指导方针
  • 将业务服务与IT相结合
  • 创建一个框架（4P）
    • 人员（people）
    • 流程（Proceses）
    • 技术/产品（Product）
    • 合作伙伴/供应商关系（Partner）
  • 提高效率及成本效益、降低风险
  • 简化IT服务的质量管理

# ITIL的定义

  • ITIL是一个公开的良玉（Domain）
  • ITIL = IT Infrastructure Library
    • 一个IT服务管理的框架
    • 建立“良好的习惯做法”
    • 定义一个IT服务之狼的标准，用于客户提出服务要求，及提供者来满足要求
  • ITIL 是管理科学子啊信息技术中的应用，是一种基于流程的方法。

# ITIL 的显著特征

  • 一客户为导向，给客户提供高质量、低成本的IT服务
    • 向客户请教并帮助他们使用优质的服务
    • 收据并分析用户的观点和建议
    • 对用户不满意的地方进行跟踪
    • 监控用户对服务的评价
    • 支持内部用户

# ITIL 的历史

  • 英国商务部拥有和负责维护ITIL
    • version 1
      • 由英国政府中央计算机与电信管理中心（CCTA）开发
      • 发表于80年代，约40本书
    • version 2
      • 发布于90年代，将40本书浓缩为2本
        • Service Delivery
        • Service Support
    • version 3
      • 发布于2007.7，包括5本书
      • 更新与2011年，ITIL 2011
      • OGC关于ITIL2011 的说明，对ITIL V3属于的更新和一致，便于理解
  • ITIL 标准被广泛认可
    • 最初的标准BS15000-1 和 BS-15000-2
    • ISO/IEC 20000-1 和 ISO/IEC 20000-2

# IT服务管理相关人员

  • 战略

    • 决策管理者
    • 外部顾问
    • 内部IT顾问

  • 战术

    • 项目经理，服务经理，IT程序经理
    • 安全官员，会计，审计，采购

  • 运营

    • 内部和外部服务提供者
    • 软件开发人员，软件提供商

# ITIL V3 相关团体

  • OGC ITIL 版权拥有者
    • APM Group ITIL 认证审批机构
      • ITIL推广者，ITIL 认证培训机构
