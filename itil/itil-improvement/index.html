# 序 前言

持续服务改进

持续服务改进（CSI），主要负责IT服务管理流程和IT服务本身的改进，持续度量和改进IT服务提供者的性能。

  • 改进流程，IT服务，IT基础架构
  • 提高效用和效率以及成本效益

# 生命周期

# CSI的目的

  • 实战保持IT服务于业务变化对齐
  • 实施IT服务改进，来支持业务流程
  • CSI负责寻找方法来改进流程的效力和效率，并同时关注成本效益

# 基线

  • 一个标记或者起始点，用于后来进行对比
  • 在开始改进一个服务的时候，必须知道现有情况
  • 用作一个厨师的数据点，来进行服务改进

# 基线的要求

  • 必须被文档化，识别和接受
  • 必须在如下每个级别上建立
    • 战略目标
    • 战术流程成熟
    • 运作指标和KPI

# 三类度量指标

  • 技术指标
    • 组件和应用指标，如：性能、可用性
  • 流程指标
    • 关键成功银子
    • 关键性能指标
    • 回答如下领域问题
      • 质量、性能、价值、合规
  • 服务指标
    • 度量点对点服务的结果

# 风险

  • 范围过大，定义的范围过大导致风险（交不了作业）
  • 不予业务讨论改进机会
  • 不同步关注改进服务和服务管理流程
  • 没有改进项目优先级
  • 没有技术或很少技术实施CSI
  • 没有资源
  • 没有知识传递及培训
  • 没有完整执行7个步骤
