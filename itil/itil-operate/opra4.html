# 第五章：请求实现概述

# 什么是服务请求 & 请求实现?

来自用户的，对信息，建议，标准变更的请求，或者访问IT服务的请求，服务请求有如下几个特点

  • 低成本
  • 低风险
  • 低技术含量 & 频繁发生

请求实现是负责管理所有服务请求什么周期的流程

# 目的与目标

  • 提供用户请求和接受标准服务的渠道
  • 为客户提供信息或者软件介质、许可证等标准服务

# 价值

  • 标准服务的快速、有效风味，提升服务质量
  • 标准服务的集中提供，便于管理和提升，节约成本

# 请求角色

  • 通常没有专属的人员
    • 服务台人员
    • 事故管理人员
    • 服务运营团队

# 请求实现的困难

  • 价值呈现方式 = 用户（用户觉得你帮我处理上网问题就是你的价值）
    • 通过技术分享打破用户的认知
    • 通过已有的文档知识库，让其自己解决
  • IT人员的认知：这是个不紧急不重要的问题（所以网管还是招服务态度和意识好的）

# 请求实现挑战

  • 定义的规范不当，用户不清楚具体有哪些请求流程
    • 请求目录（例）
      • 服务器请求
      • 账号请求
  • 用户接口设计或实施不当，是的用户很难提出需求
  • 后端履行流程的设计或操作很差，不能处理打的请求里，或某些特性的请求
  • 监视功能不足，是的无法收集准确的指标
