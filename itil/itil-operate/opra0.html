# 第一章：服务台

# 服务台的定义

  • 服务提供者与用户管理之间的唯一联系点（SPOC）
  • 管理事故和服务请求
  • 处理与用户之间的沟通

# 服务台的作用

  • 在客户、用户、IT服务和第三方支持组织指尖的日复一日的重要的联络点
  • 为所有呼叫提供一个重要的中心联络点
  • 使正常操作服务更易恢复，在议定的服务优先级里对客户的业务影响最小
  • 生成供交流和分析的报告
  • 为组织提供价值

# 服务台的功能

# Call Center

  • 接受请求

# Help Desk

  • 接受请求
  • 处理
    • 文档
      • 提供（一线，二线）
      • 操作手册
      • 常见孤战

# Service Desk

  • 接待
  • 处理
  • 日常运维
    • 机房
    • 系统
    • 应用

# 服务台消息路由

# 服务台的主要动作

  • 接听电话和接受邮件申告
  • 记录
  • 分类
  • 判断优先级
  • 升级
  • 寻找解决方案
  • 更新
  • 为其他ITIL流程提供信息
  • 检查CMDB的准确性
  • 提供报告

# 服务台类型

  • 本地服务台
  • 集中服务台
  • 虚拟服务台
  • 日不落服务台（海外）

# 本地服务台

  • 在每个不同地区建立自己的子服务台，在IT服务管理起始阶段存在的一种形式，旺旺这是企业仅在一个地点拥有办公地点
  • 有点事支持本土化的服务，企鹅电视容易引起重复建设从而增加成本

# 集中服务台

由一个服务中心集中处理所有的服务请求，适用于当IT部门同时负责it服务和it服务的支持。

# 虚拟服务台（阿里云）

虚拟式服务台

根据结构和目标，服务台的人力设置也不同。对只记录和转移呼叫的，可以由语音系统实现；技能型服务台可以结合知识库处理事件；专家型服务台需要专业it知识并能独立解决大部分突发性事件。

# 服务台结构选择实践

  • 业务状态考虑
    • 业务的服务对象
    • 业务的规模、对服务质量的要求
    • 业务的分布式情况
  • 技术考虑
    • 系统工具的集成，或者所有的服务台共享一套系统
    • 语音的路由
  • 流程考虑
    • 必须建立统一的管理流程，及专门的管理人员
  • 实施考虑：集中式服务台 + 现场服务台（驻场）
    • 集中的服务台记录用户的事故及请求
    • 远程提供支持
    • 远程无法解决，派遣至现场服务

# 服务台考核及外购

  • 服务台的度量指标必须切合实际
  • 量化
    • 一线解决率
    • 解决事故平均时间
    • 升级事故平均时间
    • 处理事故平均成本
    • 平均回顾和关闭解决的事件
    • 客户满意度调查
      • 电话
      • 个人约见（阿里团队咨询）

# 外购

  • 对服务台提供的活动和服务的责任不能外购
  • 必须与组织的其他团队和部门有效和高效合作
  • 维护端到端的服务管理控制
    • 通用的工具和流程
    • SLA 目标
    • 沟通
    • 数据的所有权

# 服务台角色

  • 服务台经理

    • 全面负责服务台的事故和服务处理请求
    • 作为升级的点
    • 向高级经理报告可能影响业务的问题
    • 参加变更顾问委员会会议

  • 服务台分析师

    • 接电话，提供一线支持，处理事故或服务请求

  • 超级用户

    • 业务用户，作为IT与服务台之间的联系点
    • 使得IT与业务指尖的沟通在操作层面变得容易

# 服务台成功的关键因素

  • 理解业务需求
  • 理解客户需求
  • 投资与对客户、支持团队和志愿的培训
  • 服务级别是实际有效，协商一致和有规律地回顾
  • 其收益能够被业务部门所接受

# IT运营管理

  • 执行组织的日常运营活动
    • 执行再用的活动和过程
  • 主要工作
    • 运营控制
      • 见识和监控IT基础设施中的事件和运营活动
        • 控制塔管理、工作调度、备份恢复、性能维护活动
    • 设备管理
      • 管理物理的IT环境
        • 数据中心、恢复中心、供电、空调

# 文档和角色

  • 文档

    • 标准操作程序（SOP）
    • 运营日志
    • 轮班计划和报告
    • 运营轮班

  • 角色

    • IT运营经理

    • 轮班Leaders

    • IT运营分析师

    • 自身IT运营分析师

    • IT操作员

# 技术管理

# 概述

提供技术专家意见，负责IT基础设施的技术管理

  • 小组
  • 部门
  • 团队

# 角色

  • IT基础设施相关的技术知识和经验管理人

  • 提供实际的资源，来支持IT服务管理生命周期

  • 目标

    • 帮助计划、实施和维护稳定的技术架构

# 文档

  • 技术文档

    • 技术手册
    • 管理手册
    • 用户手册

  • 维护计划

  • 技能库

  • 角色

    • 技术经理团队Leader
    • 技术分析师/架构师
    • 技术操作员 - 执行技术管理中日常的运营任务

# 应用管理

# 概述

  • 在生命周期内，管理应用
    • 部门，小组或团队来执行
    • 通常不是应用开发团队
  • 职责
    • 提供如何管理应用的知道
    • 协助应用设计
    • 帮助是否购买或开发应用的决策
  • 目标
    • 实现日常流程和活动的稳定
    • 改进服务，节约成本
    • 快速和解决IT运营故障

# 构建还是购买

  • 应用程序管理在服务设计阶段就协助决定
    • 应用程序的规模和负载
    • 管理需求
    • 识别运营成本
    • 报表或集成的数据访问需求
    • 研究现有的工具可以满足功能到什么程度，以及需要基于此作多大的定制
    • 评估定制的成本
    • 识别支持解决方案所需要的技能
    • 工具管理需求
    • 安全需求
