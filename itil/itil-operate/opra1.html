# 第二章：事件管理

# 事件

  • 任何可被检测或者辨别、对IT基础架构及IT服务交付的事情

# 事件管理目的

  • 检测和感知事件，并确定对应的控制行动
  • 自动启动其他流程，作为服务运营流程的切入点

# 事件管理范围

  • 配置项
  • 环境条件
  • 软件许可监控
  • 安全
  • 正常活动

# 监视工具

  • 主动监视工具
  • 被动监视工具

#

  • 事故的早期检测
  • 实时和自动异常报告
  • 主动恢复活动
    • 某些情况下，可以实现基础架构的自我恢复，或者对业务的影响
  • 集成到服务管理
    • 事件工具可以通知合适的人，记录事故日志
    • 如果集成到能力和可用性管理，工具可以自动解决问题

# 流程

# 重要性（监控级别）

  • 信息
  • 告警
  • 异常

# 挑战

  • 获取自己和资源去购买及部署监控工具，以发挥监控系统的有事
  • 设定正确的过滤级别
  • 自动的监控活动会产生额外的网络流量，这可能对网络容量级别产生消极效果
  • 必要技能的获取时且成本高
  • 在没有设置运维流程的情况下部署事件管理工具
